## Refined Phase Decomposition (5 invocation phases, exit-safe)

| # | Phase     | Reads                | Writes                                                      | Subagents |
|---|-----------|----------------------|-------------------------------------------------------------|-----------|
| P1| decompose | user query           | {slug}-plan.md, state.phase=search                          | none      |
| P2| search    | plan, state, findings/| new findings file(s), state.completed_subtopics, phase     | 1-3 [omh-role:researcher] parallel |
| P3| gap_check | all findings         | optional _followup.md OR direct phase flip                  | 0 or 1 [omh-role:researcher] |
| P4| synthesize| all findings (parent inlines) | {slug}-report.md status: draft, phase=verify         | 1 [omh-role:research-synthesist] |
| P5| verify    | report+findings (parent inlines) | confirmed frontmatter / state mutate / blocked     | 1 [omh-role:research-verifier] |

Re-entrant: P2 runs once per batch of ≤3 subtopics until done. Singleton
state file (omh_state mode="research") = at most one active session per
project (matches omh-deep-interview line 233 idiom). Cancellation checked at
every phase entry.

Boundary rule (uniform, per C1): PARENT owns the filesystem. Parent reads
findings files and INLINES their contents into synthesist's and verifier's
delegation `context` field. Subagents return structured text only and never
touch the filesystem (no read, no write).

## Roles to Create (unchanged from R1, with explicit boundary updates)

1. role-researcher.md (~30 lines): one subtopic, returns SUBTOPIC /
   QUERIES_RUN / SOURCES (with credibility tags) / SYNTHESIS / GAPS.
   Empty-result sentinel rule (per C3): if no usable sources, returns
   well-formed block with `SOURCES: (none-found, reasons listed)` and
   `SYNTHESIS: (insufficient sources for this subtopic)`. Returns text only;
   parent writes the file.

2. role-research-synthesist.md (~30 lines): reads inlined findings from its
   `context` field, returns markdown report (Exec Summary, Detailed Findings,
   Key Takeaways, Open Questions, Sources [N]). Frontmatter `status: draft`.
   No filesystem access. Per C3: must propagate `(insufficient sources)`
   sections verbatim, MUST NOT fabricate.

3. role-research-verifier.md (~30 lines): READ-ONLY (prose contract; A5
   Known Gap if Hermes lacks per-call tool scoping). Cross-checks every [N]
   against inlined findings/sources. Iron Law section adapted from
   role-verifier.md, retargeted at citations. Per C3: explicit rule —
   `(insufficient sources)` sections are NOT verification failures; they
   become Open Questions in the report.

      - Set `state.phase = blocked`.

      - Log `BLOCKED_RETRIES_EXHAUSTED slug={slug}`.
